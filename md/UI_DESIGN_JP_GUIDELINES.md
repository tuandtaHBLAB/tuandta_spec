# UI Design Guidelines For Japanese Exam Screens

## Goal
Create UI that feels simple, modern, calm, and trustworthy for Japanese users.
The design should support concentration first, decoration second.

## Core Principles
- Prefer clarity over novelty.
- Keep one visual focus per screen.
- Make the reading order obvious at a glance.
- Use generous spacing so the screen feels calm, not crowded.
- Remove decorative elements that do not improve comprehension.

## Visual Tone
- Use light backgrounds with soft contrast.
- Prefer clean blues, slate, white, and muted neutrals.
- Avoid overly playful gradients, loud pink accents, or flashy effects unless there is a strong reason.
- Shadows should be soft and shallow. Borders should be subtle and crisp.

## Typography
- Japanese text must be easy to scan and stable in rhythm.
- Headlines should be strong but not aggressive.
- Body text should prioritize line height and readability over visual impact.
- Furigana should be clearly smaller than base text, but still readable.
- Do not mix too many font moods on one screen.

## Layout
- Top area should establish context quickly: part name, timer, or screen purpose.
- Main content should sit in a single clear content block.
- Actions should be obvious and easy to reach.
- Keep supporting metadata secondary and quiet.

## Buttons
- Primary actions should look dependable and calm.
- Use one strong primary CTA per screen.
- Avoid candy-like buttons or exaggerated glow.
- Button labels should be direct and short.

## Reading Screens
- Long Japanese reading passages should appear inside a clean white card.
- Keep line length comfortable.
- Use larger line height for reading tasks.
- Do not crowd the passage with extra labels unless they help the task.
- Furigana layouts must remain visually steady across lines.

## Exam Screens
- Timers should be visible but not dominate the screen.
- Camera preview, audio meter, and anti-cheat UI should feel functional and quiet.
- Do not let floating utility UI distract from the main exam content.

## Mobile Behavior
- Preserve the same reading hierarchy as desktop.
- Expand content vertically instead of shrinking text too aggressively.
- Keep CTA width large enough for confident tapping.

## Avoid
- Over-decoration
- Excessive color contrast
- Dense card stacking
- Multiple competing focal points
- Playful UI patterns that reduce trust in an exam context

## Implementation Rule
Before changing any exam UI, read this file and check whether the screen is:
- calm
- readable
- task-first
- visually balanced
- culturally natural for Japanese users
